# Data

The experiments operate on per-prediction dumps from an EfficientDet-d0
detector trained with loss attenuation. Each line of a dump is a Python
literal (parsed with `ast.literal_eval`) describing one detection on the
validation set. The fields used here are:

| key                            | meaning                                                                |
| ------------------------------ | ---------------------------------------------------------------------- |
| `bbox`                         | predicted bounding box `[ymin, xmin, ymax, xmax]` (pixels)             |
| `gt_bbox`                      | matched ground-truth bounding box                                      |
| `gt_class`                     | ground-truth class index                                               |
| `probab`                       | per-class softmax probabilities                                        |
| `uncalib_albox` (KITTI / CODA) | per-coordinate aleatoric std-dev from loss attenuation (raw)           |
| `uncalib_uncert_albox` (BDD)   | same as above, named differently in BDD dumps                          |
| `iso_all_albox`                | aleatoric std-dev calibrated by global isotonic regression (Rel. IR)   |
| `rel_iso_perclscoo_albox`      | aleatoric std-dev calibrated per-coordinate / per-class (Rel. IR PCoPC) |

## Files

| File                          | Status   | Contents                                                                   |
| ----------------------------- | -------- | -------------------------------------------------------------------------- |
| `prediction_data_val_unc.txt` | included | KITTI validation predictions (1589 detections).                            |
| `validate_results_CODA.txt`   | included | CODA predictions from the BDD-trained model (4931 detections).             |
| `validate_results_BDD.txt`    | **missing** | BDD validation predictions (~37k detections). Too large to ship in the repository. |

## Reproducing `validate_results_BDD.txt`

The dump is generated by the EfficientDet-d0 + loss-attenuation pipeline from
the companion repository
[mos-ks/uncertainty-detection-autolabeling](https://github.com/mos-ks/uncertainty-detection-autolabeling/tree/main).
After training and calibrating the BDD model, run the validation dump step
described there and copy the resulting file into this directory under the
expected name.
